Kadence - Public API - Examples

This page showcases a selection of examples demonstrating the use of our public API. While not comprehensive, these examples serve as a starting point to inspire and guide you in integrating the API into your own tools and applications.

A good starting point before starting use the API is our getting started guide and our API specification.

All examples in this project are deliberately light-weight with as much limited boilerplate as possible. All examples are commented with explanatory text where appropriate with links to the Kadence API documentation.

Note: If you are building a web application you should make all API calls server-side with an abstraction layer so that you do not expose the API key or give API access to the full API to all viewers of the webpage. For demonstration purposes we have made all calls 'client-side' (via a proxy) to more easily demonstrate how to call and interact with the API. If you do this in production you will give all visitors to your webpage full access to the Kadence API. This is not recommended.

List Bookings

One of the most basic things you can do with the Kadence API is to retrieve bookings information. In this example we will showcase:

  • Retrieving and listing bookings.
  • Retrieving information about buildings.
  • Retrieving users information.
